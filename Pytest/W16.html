Comment Cleanup Rules

1. Remove all debug print statements and commented-out code
2. Delete vague TODOs (or make them specific with issue numbers)
3. Add docstrings to all functions explaining purpose, args, and returns
4. Keep inline comments for non-obvious logic only

üóëÔ∏è Dead Code Removal

Dead code is unused functions, imports, variables, or commented-out blocks. It confuses readers and can introduce security risks.

Finding Dead Code with Python Tools
# Install vulture (dead code finder) $ pip install vulture # Scan your project $ vulture src/ --min-confidence 80 src/utils.py:45: unused function 'old_parser' (90% confidence) src/main.py:12: unused import 'requests' (100% confidence) src/analyzer.py:78: unused variable 'temp_data' (80% confidence) # Also check for unused imports with flake8 $ pip install flake8 $ flake8 src/ --select=F401 src/main.py:3:1: F401 'os' imported but unused
‚ö†Ô∏è

Common Dead Code Patterns

‚Ä¢ Imports you tried but never used
‚Ä¢ Functions from early iterations
‚Ä¢ Commented-out "backup" code
‚Ä¢ Debug variables left behind

Keep in Version Control

üì¶ Dependency Management

Hidden dependencies are the #1 reason projects fail on other machines. Here's how to ensure yours works everywhere:

Creating requirements.txt
# Option 1: Freeze current environment (includes everything) $ pip freeze > requirements.txt # Option 2: Better - use pipreqs (only includes imports you use) $ pip install pipreqs $ pipreqs src/ --force # Verify by installing in fresh environment $ python -m venv test_env $ source test_env/bin/activate # or test_env\Scripts\activate on Windows $ pip install -r requirements.txt $ python src/main.py --help # Should work!
Example Good requirements.txt
# Core dependencies requests>=2.28.0 python-dateutil>=2.8.0 # Data processing pandas>=1.5.0 # Testing (dev dependency) pytest>=7.0.0
