Module 1: Installation & Setup

Let's get your Python development environment ready. Choose your operating system:

Installing Python 3.x on Windows

Download Python
Navigate to python.org/downloads and download the latest Python 3.x installer for Windows (64-bit recommended).
Run the Installer
CRITICAL: Check "Add Python to PATH" before clicking "Install Now". This allows you to run Python from any command prompt.
‚ö†Ô∏è Common Mistake
Forgetting to check "Add Python to PATH" is the #1 reason students have issues. If you forgot, uninstall and reinstall with this option checked.
Verify Installation
Open Command Prompt (Win + R, type "cmd", press Enter) and verify Python is installed:
C:\Users\student> python --version
Python 3.12.0
C:\Users\student> pip --version
pip 23.3.1 from C:\Python312\lib\site-packages\pip (python 3.12)

Installing VS Code

Download VS Code
Visit code.visualstudio.com and download the Windows installer.
Install with Recommended Options
During installation, check these options:
  • Add "Open with Code" action to Windows Explorer file context menu
  • Add "Open with Code" action to Windows Explorer directory context menu
  • Register Code as an editor for supported file types
  • Add to PATH
Launch and Verify
Launch VS Code. You should see the welcome screen. You're ready for extensions!

Essential Extensions

Installing Python 3.x on macOS

Check Existing Python
macOS comes with Python 2.7 (outdated). Open Terminal and check:
student@macbook ~ % python3 --version
zsh: command not found: python3
If you see "command not found", you need to install Python 3.
Install via Homebrew (Recommended)
First, install Homebrew if you haven't:
student@macbook ~ % /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Then install Python 3:
student@macbook ~ % brew install python3
Verify Installation
student@macbook ~ % python3 --version
Python 3.12.0
student@macbook ~ % pip3 --version
pip 23.3.1 from /opt/homebrew/lib/python3.12/site-packages/pip (python 3.12)

Installing VS Code

Download VS Code
Visit code.visualstudio.com and download the macOS version.
Install the Application
Open the downloaded .zip file, drag Visual Studio Code to your Applications folder, then launch it.
Add to PATH (Optional but Recommended)
In VS Code, press Cmd+Shift+P, type "shell command", and select "Install 'code' command in PATH". This lets you open files from terminal with code filename.py.

Essential Extensions

Python Extension
Press Cmd+Shift+X to open Extensions. Search for "Python" by Microsoft and click Install.
Git Extension
macOS includes Git by default. Open terminal and type git --version to verify. VS Code will detect it automatically.
GitHub Copilot (Optional)

Installing Python 3.x on Linux

Check Existing Python
Most Linux distributions include Python 3. Check your version:
student@kali:~$ python3 --version

student@kali:~$ python3 --version

student@kali:~$ sudo apt-get install -f
For .rpm packages: sudo rpm -i code-*.rpm
Launch VS Code
Type code in terminal or find it in your applications menu.

Module 2: Running Python

Now that your environment is set up, let's write and execute your first Python program.

Creating hello_world.py

Create Project Folder
Open VS Code, press Ctrl+K Ctrl+O (or Cmd+K Cmd+O on Mac), and create a new folder called "python_basics".
Create New File
In VS Code, press Ctrl+N (Cmd+N on Mac) to create a new file. Save it as hello_world.py in your python_basics folder.
Write Your First Program
Type this code in your file:
hello_world.py
1 # My first Python program - Cybersecurity Edition
2 print("Hello, Cyber World!")
3 print("Initializing security protocols...")
üí° Code Breakdown
  • Line 1: Comments start with # and are ignored by Python
  • Line 2-3: print() displays text to the console
  • Strings: Text wrapped in quotes ("..." or '...')

Running from VS Code

Using the Run Button
With hello_world.py open, click the "Run" button (‚ñ∂Ô∏è) in the top-right corner of VS Code, or press Ctrl+Alt+N.
OUTPUT
Hello, Cyber World!
Initializing security protocols...
Using F5 (Debug Mode)
Press F5 to run in debug mode. This allows you to:
  • Set breakpoints (pause execution at specific lines)
  • Inspect variable values
  • Step through code line by line
We'll explore debugging in detail later.
Right-Click Run
Right-click anywhere in your code and select "Run Python File in Terminal". This opens an integrated terminal and runs your script.

Running from Terminal

Open Terminal in VS Code
Press Ctrl+` (Cmd+` on Mac) to open the integrated terminal, or use Terminal ‚Üí New Terminal from the menu.
Navigate to Your Script
Use cd (change directory) to navigate to where your script is saved:
$ cd python_basics
$ ls
hello_world.py
Execute the Script
Run your Python script with the python command:
$ python hello_world.py
Hello, Cyber World!
Initializing security protocols...

Module 3: Errors & Debugging

Errors are not failures‚Äîthey're feedback. Understanding error messages is crucial for becoming an effective cybersecurity developer.

Understanding Python Errors

When Python encounters an error, it provides a traceback‚Äîa detailed report showing what went wrong and where. Learning to read tracebacks efficiently is essential.

üî¥ Common Error Types
  • SyntaxError: Code doesn't follow Python's grammar rules
  • NameError: Variable/function name doesn't exist
  • TypeError: Operation on wrong data type
  • IndentationError: Incorrect spacing/indentation
  • AttributeError: Object doesn't have the requested attribute
  • ImportError: Module can't be found or imported

Anatomy of a Traceback

Let's dissect a traceback to understand each component:

port_scanner.py
1 # Buggy port scanner
2 import socket
3
4 target = "192.168.1.1"
5 port = "80" # BUG: Should be int, not string
6
7 sock = socket.socket()
8 result = sock.connect_ex((target, port))

When you run this code, Python gives you this traceback:

ERROR TRACEBACK
Traceback (most recent call last):
File "port_scanner.py", line 8, in <module>
result = sock.connect_ex((target, port))
TypeError: an integer is required (got type str)
üîç Breaking It Down
  • Traceback (most recent call last): Header indicating error trace
  • File "port_scanner.py", line 8: Where the error occurred
  • result = sock.connect_ex(...): The problematic code line
  • TypeError: The error type
  • an integer is required (got type str): What went wrong‚Äîport should be int, not string

Common Errors in Action

1. SyntaxError

syntax_error.py
1 print("Checking firewall status..."
2 # Missing closing parenthesis above!
File "syntax_error.py", line 2
# Missing closing parenthesis above!
^
SyntaxError: invalid syntax

2. NameError

name_error.py
1 password_hash = "5f4dcc3b5aa765d61d8327deb882cf99"
2 print(pasword_hash) # Typo: "pasword" not "password"
Traceback (most recent call last):
File "name_error.py", line 2, in <module>
print(pasword_hash)
NameError: name 'pasword_hash' is not defined

3. IndentationError

indentation_error.py
1 def scan_network():
2 print("Starting scan...")
3 print("Scan complete") # Inconsistent indent
File "indentation_error.py", line 2
print("Starting scan...")
^
IndentationError: expected an indented block
