‚úì Capstone Project Requirements

‚ö†Ô∏è NON-NEGOTIABLE: All capstone projects MUST include these elements. No exceptions!

üéØ Mandatory Technical Requirements

  • At least one JSON input file: Your program must read data from a JSON file (could be config, data, rules, etc.)
  • At least one JSON output file: Your program must write results to a JSON file (report, log, processed data, etc.)
  • File system interaction: Read from and/or write to disk beyond just JSON (logs, reports, data files)
  • CLI interface with arguments: Use argparse or similar to accept command-line parameters
  • At least one class used meaningfully: Not just a wrapper‚Äîa class that models your domain
  • Basic tests for core logic: Unit tests using unittest or pytest for key functions
  • Logging and error handling: Use Python's logging module and handle errors gracefully

üìè Scope Guidelines

‚úÖ Just Right Scope

  • 200-400 lines of meaningful Python code (excluding tests)
  • 3-5 core features that work well together
  • Can be completed in 2-3 weeks with steady effort
  • Solves a specific, well-defined problem
  • Has clear inputs and outputs

‚ùå Too Big (Reduce Scope)

  • Building a full SIEM system
  • Creating an enterprise-grade application
  • Requires external databases or complex infrastructure
  • Would take months to implement properly

‚ùå Too Small (Expand Scope)

  • Just a single function or script
  • No real file system interaction
  • Could be completed in a few hours
  • Doesn't combine multiple concepts

üí° Effective AI Prompts for Each Stage

Stage 1: Brainstorming & Idea Selection

I'm a cybersecurity student choosing a capstone project. I'm interested in [YOUR INTEREST AREA: log analysis / malware detection / network security / etc.]. The project must: - Be completed in 2-3 weeks - Use Python with JSON input/output - Be 200-400 lines of code - Solve a real IT/security problem Can you suggest 3 project ideas that: 1. Match my interest area 2. Meet these constraints 3. Would be valuable for a portfolio For each idea, explain: what it does, why it's useful, and what makes it interesting.

Stage 2: Scope Refinement

I want to build [YOUR PROJECT IDEA]. Here's my initial plan: [PASTE YOUR DESCRIPTION] Please help me evaluate: 1. Is this scope appropriate for a 2-3 week student project (200-400 lines)? 2. Is it too ambitious? If so, what features should I cut? 3. Is it too simple? If so, what would make it more substantial? 4. What are the core features (must-haves) vs. nice-to-haves? Give me a realistic "MVP" version that still demonstrates strong Python skills.

Stage 3: JSON Schema Design

A JSON schema for my output/report file 3. Explanation of why this structure makes sense Consider: - The data should be easy to parse in Python - The output should be useful for [YOUR TARGET USER] - Include all necessary fields for [YOUR ANALYSIS]

Stage 4: Architecture & Class Design

Requirements: - Read JSON input files - Process data to [YOUR GOAL] - Write JSON output files - Use at least one meaningful class - Have a CLI interface Can you suggest: 1. What classes I should create (and why) 2. What key methods each class should have 3. How the classes should interact 4. What the main program flow should look like Keep it simple and educational - I need to understand and implement this myself!

Stage 5: Proposal Review

I've written my capstone proposal. Can you review it and provide feedback? [PASTE YOUR PROPOSAL] Please evaluate: 1. Is the problem statement clear and compelling? 2. Are the inputs/outputs well-defined? 3. Is the scope appropriate (not too big or small)? 4. Are the must-have features realistic for 2-3 weeks? 5. Is the technical approach sound? 6. What's missing or unclear? Be constructive but honest - I want to submit a strong proposal!

‚ö†Ô∏è AI Pitfalls to Avoid

  • Don't accept the first answer: Ask follow-up questions, request alternatives
  • Don't trust blindly: Verify technical claims, test suggestions
  • Don't skip thinking: AI should enhance your design process, not replace it
  • Don't copy-paste: Understand every line of code you include
  • Don't over-complicate: Simpler solutions are often better

üéØ Best Practice: Iterative Refinement

Have a conversation with AI, don't just ask one question:

  1. Ask your initial question
  2. Review the response critically
  3. Ask follow-ups: "What about X?" or "How would you handle Y?"
  4. Challenge assumptions: "Is there a simpler way?"
  5. Request alternatives: "What's another approach?"
